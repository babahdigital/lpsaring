# Worklog 2026-02-23 — Standardisasi UI (Vuexy-like)

Dokumen ini merangkum pekerjaan UI yang dikerjakan bertahap untuk membuat tampilan **selaras** (admin & user), **responsif**, dan **minim styling ad-hoc**.

## Tujuan

1. Rapikan UI “Tunggakan Kuota” (spacing presisi, tombol konsisten, estimasi rata kanan).
2. Samakan pola **table/datatable/select/pagination** mengikuti gaya demo Vuexy.
3. Perbaiki isu mobile:
   - Admin Packages tidak muncul di mobile.
   - Admin Users mobile tidak punya pagination.
4. Tahap lanjutan: tambah **datatable toolbar** reusable (entries selector + search layout ala demo).

## Perubahan yang dilakukan (ringkas)

### 1) Debt list: spacing & layout

- Akar masalah jarak “jauh sekali” berasal dari rule global `.card-list .v-list-item:not(:last-child)` yang memberi jarak tambahan.
- Solusi: dibuat varian khusus list untuk kasus debt agar spacing rapat tanpa merusak list lain.

File terkait:
- frontend/@core/scss/base/libs/vuetify/_overrides.scss
- frontend/pages/dashboard/index.vue
- frontend/pages/riwayat/index.vue

Hasil:
- Tombol “Lunasi” presisi di kanan.
- “Estimasi: Rp …” nilai rapi dan rata kanan.
- Di Riwayat tinggal satu tombol (hapus yang duplikat).

### 2) Standardisasi table/datatable/select/pagination

Pola utama yang distandarkan:

- Datatable footer bawaan **dimatikan** agar tidak beda-beda style antar halaman (`hide-default-footer`).
- Pagination diganti menjadi **1 komponen** (`TablePagination`) di semua halaman.
- Select distandarkan via wrapper (`AppSelect`) agar density/variant/hide-details konsisten.
- Style table dipindah ke global SCSS supaya mirip Vuexy dan tidak perlu styling ulang per halaman.

File terkait:
- frontend/@core/components/TablePagination.vue
- frontend/@core/components/app-form-elements/AppSelect.vue
- frontend/@core/scss/template/libs/vuetify/components/_table.scss

Halaman/komponen yang di-update (contoh utama):
- Admin: packages, users, transactions, requests, logs, promos, whatsapp, backup, settings/notifications
- User: requests, riwayat

### 3) Fix bug responsif (mobile)

- Admin Packages: pada mobile sebelumnya tidak menampilkan list sebenarnya (terlihat seperti “tidak muncul”).
- Admin Users: pada mobile sebelumnya tidak ada pagination.

Hasil:
- Daftar Packages tampil di mobile dan tetap bisa dipaging.
- User list di mobile punya pagination.

### 4) Tahap lanjut: toolbar datatable reusable (entries + search)

Ditambahkan komponen toolbar reusable agar layout “Show entries” dan “Search” konsisten seperti demo.

File terkait:
- frontend/@core/components/DataTableToolbar.vue

Integrasi awal:
- Admin Transactions: migrasi penuh (export action di slot kiri toolbar, search di kanan).
- Halaman lain: ditambahkan entries selector (search dipindah bila aman/masuk akal) agar konsisten.

## Error/masalah yang ditemui & penyelesaian

1. **Spacing list terlalu besar**
   - Penyebab: override global `.card-list` memberi gap besar.
   - Solusi: buat class khusus debt list (`.debt-card-list`) agar tidak mengubah semua list.

2. **Admin Packages kosong di mobile**
   - Penyebab: mobile view tidak merender data list yang sebenarnya.
   - Solusi: implement mobile list + pagination.

3. **Pagination hilang di beberapa layar**
   - Penyebab: footer bawaan datatable dimatikan tetapi pagination custom belum dipasang untuk semua variasi (desktop/mobile).
   - Solusi: gunakan `TablePagination` sebagai single source.

4. **Template Vue invalid (dua `v-else` berurutan)**
   - Penyebab: saat menambah toolbar di Riwayat desktop, sempat ada dua `v-else` sibling.
   - Solusi: bungkus toolbar+table dalam satu blok `v-else` yang sama.

## Dampak ke pola pengembangan ke depan

- UI table tidak lagi perlu “styling per halaman” untuk hal yang generik.
- Halaman baru cukup mengikuti pola:
  1) `DataTableToolbar` (opsional search)
  2) `VDataTableServer` + `hide-default-footer`
  3) `TablePagination`
