## Context

Sesi ini menutup dua gap yang masih tersisa setelah rollout frontend/backend sebelumnya:

1. direct-browser OTP masih bisa jatuh ke `/login/hotspot-required` dengan `identity_hint_missing`,
2. user yang ter-block otomatis oleh `policy.block_transition:sync_usage` karena `quota_debt_limit` belum menerima WA khusus debt-limit.

Selain implementasi kode, sesi ini juga mencakup push, CI, docker publish, deploy, dan verifikasi runtime produksi.

## Temuan Root Cause

### 1. Direct-browser OTP gagal karena target bridge salah

Audit request produksi menunjukkan pola berikut:

- flow yang berhasil selalu lebih dulu menghantam root router `http://login.home.arpa/`,
- root router itu menjalankan `login.html` MikroTik yang kemudian me-return user ke `/captive` dengan placeholder/context hotspot yang authoritative,
- flow direct-browser yang gagal justru menembak `http://login.home.arpa/login`, sehingga context router yang dibutuhkan untuk recovery tidak pernah terbentuk.

Kesimpulan:

- manual hotspot login masih boleh memakai `/login`,
- tetapi silent bridge/probe untuk recovery context harus menjaga root router, bukan memaksa semua target lokal menjadi `/login`.

### 2. Auto debt-limit block belum punya hook WA user khusus

Audit runtime dan DB menunjukkan:

- low-quota WA sebelumnya memang sudah pernah terkirim pada sebagian user terdampak,
- block otomatis oleh `policy.block_transition:sync_usage` tetap mencatat `blocked_reason=quota_debt_limit|...`,
- tetapi jalur tersebut belum memanggil template WA khusus debt-limit ke user.

Akibatnya, user bisa melihat status blocked karena debt-limit tanpa menerima pesan penjelasan debt-limit yang sesuai event tersebut.

## Perubahan Kode

### Frontend: target bridge menjaga root router

File utama:

- `frontend/utils/hotspotLoginTargets.ts`
- `frontend/tests/hotspot-login-targets.test.ts`

Perubahan:

- ditambahkan `normalizeHotspotBridgeUrl(raw)`,
- `resolveHotspotBridgeTarget(...)` sekarang menjaga root router untuk silent bridge,
- manual login target tetap bisa dinormalisasi ke `/login` saat memang itu yang dibutuhkan UI fallback.

Hasil yang diinginkan:

- direct-browser OTP yang perlu recovery context kembali memperoleh roundtrip router root -> `login.html` -> `/captive`,
- fallback manual hotspot login tidak berubah menjadi loop internal portal.

### Backend: WA khusus saat fresh auto debt block

File utama:

- `backend/app/services/hotspot_sync_service.py`
- `backend/tests/test_hotspot_sync_debt_limit.py`

Perubahan:

- ditambahkan helper `_select_reference_package_for_debt_mb(...)`,
- ditambahkan helper `_send_auto_debt_limit_block_notification(...)`,
- `_apply_auto_debt_limit_block_state(...)` kini memanggil helper tersebut hanya saat transisi block baru karena `quota_debt_limit`.

Perilaku final:

- user menerima template `user_quota_debt_blocked` secara best-effort saat block baru terjadi,
- jika user sudah blocked sebelumnya oleh auto debt yang sama, sync berikutnya tidak mengirim duplikasi dari jalur ini.
