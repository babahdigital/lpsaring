## 2) Perubahan Implementasi Terkini

### A. Backend hardening task dan auth/network flow

1) `backend/app/tasks.py`
- Hardening `clear_total_if_no_update_submission_task` agar fallback delete set-based tetap aman saat jalur truncate/constraint tidak bisa dipakai.
- `sync_unauthorized_hosts_task` diperbarui untuk menandai error router timeout/connection sebagai non-retryable yang dicatat, agar tidak memicu retry loop tanpa nilai tambah operasional.

2) `backend/app/infrastructure/http/auth_contexts/admin_auth_handlers.py` + `backend/app/infrastructure/http/auth_routes.py`
- `reset-login` disetarakan dengan `logout` untuk revokasi sesi penuh:
  - hapus refresh token milik user,
  - hapus semua `user_devices` milik user,
  - clear cookie auth/refresh pada response,
  - fallback revoke refresh cookie jika delete-all token tidak tersedia.
- Jalur route `/api/auth/reset-login` kini mengirim dependency token/device model dan helper cookie cleanup yang sama dengan logout path.

3) `backend/app/services/device_management_service.py`
- Perbaikan edge-case: `apply_device_binding_for_login(...)` kini menginisialisasi `username_08` sejak awal fungsi.
- Tujuan: mencegah `UnboundLocalError` saat `IP_BINDING_ENABLED=False` tetapi fungsi tetap melakukan cleanup hotspot host.

4) Test backend tambahan/terkait
- `backend/tests/test_tasks_clear_total_auto_clear.py`
- `backend/tests/test_tasks_unauthorized_sync.py`
- `backend/tests/test_logout_network_reset.py`
- `backend/tests/test_auth_hotspot_session_status.py`
- `backend/tests/test_device_management_service.py`
  - ditambahkan regression test khusus mode `ip_binding_enabled=False` untuk memastikan flow tidak crash.
  - ditambahkan test parity reset-login untuk memastikan revoke token/device + clear cookie berjalan.
